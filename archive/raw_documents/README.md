# Raw Documents

This folder is for archived source material that I decided to keep, but not promote into the main reading path of the repository.

The public repository should prefer Markdown summaries in `docs/` over raw PPTX, DOCX, HWPX, datasheets, chat logs, or large video files. Raw files can contain private names, emails, duplicate research material, or oversized media that does not help the main GitHub experience.

## Current Files

- `로봇 시행착오.pdf`: recovered trial-and-error PDF generated during the project.
- `리시버문제_해결방법1.text`: receiver interference notes and mitigation ideas from the project archive.

These files are kept as archive material unless a privacy or size review decides they should move into a GitHub Release or out of the public repository.

## Public Policy

- Do not add raw presentation decks, office documents, chat exports, or vendor datasheets directly to the main repository tree.
- Summarize technical decisions in Markdown and link to the related code, diagrams, photos, or demo media.
- If raw archive material is needed, publish it separately through a release or archive workflow after privacy and file-size review.
